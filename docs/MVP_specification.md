# サウナ行った MVP仕様書

## 1. アプリ概要

### アプリ名

サウナ行った

### 目的

サウナ後でもかんたんに記録できる、超シンプルなととのいログアプリを作る。

### コンセプト

サウナ記録をがんばらない。

ユーザーが細かい入力に疲れないように、手動入力を最小限にし、選択式の入力を中心にする。

## 2. MVPのゴール

MVPでは、以下ができる状態を完成とする。

- サウナ記録を追加できる
- 過去のサウナ記録を一覧で見られる
- 1件の記録を詳しく見られる
- 記録を編集できる
- 記録を削除できる
- 記録内容からX投稿文を生成できる
- X投稿画面または共有画面を開ける

## 3. 対象ユーザー

### メインユーザー

- サウナが好きな人
- サウナ記録を残したいが、細かい入力は面倒な人
- サウナ後にXへ投稿したい人
- エンジニアや在宅ワーカーなど、日常的にXを使う人

### ユーザーの悩み

- 記録アプリは入力項目が多いと続かない
- サウナ後に細かいことを入力するのがだるい
- でも後から見返せる記録はほしい
- Xに投稿する文章を毎回考えるのが面倒

## 7. X投稿仕様

## 7.1 基本方針

MVPでは、X APIによる自動投稿は実装しない。

記録内容から投稿文を自動生成し、スマホの共有機能またはX投稿画面を開く。

投稿前にユーザーが文章を編集できる状態にする。

## 7.2 投稿文に含める情報

| 項目 | 含めるか |
|---|---|
| 施設名 | 含める |
| ととのい度 | 含める |
| セット数 | 入力されていれば含める |
| サウナの種類 | 入力されていれば含める |
| メモ | 含めない |
| ハッシュタグ | 含める |

## 7.3 投稿文テンプレート

```text
サウナ行った。
今日は{ととのい度アイコン} {ととのい度ラベル}。

{施設名} / {セット数} / {サウナの種類}

#サウナ #ととのい
```

## 7.4 投稿文の例

```text
サウナ行った。
今日は🫠 ととのった。

〇〇温泉 / 3セット / ドライサウナ

#サウナ #ととのい
```

## 7.5 投稿文生成関数イメージ

```ts
const generateXPostText = (log: SaunaLog) => {
  const mood = moodOptions.find((item) => item.value === log.mood);
  const setCount = setCountOptions.find((item) => item.value === log.setCount);
  const saunaType = saunaTypeOptions.find((item) => item.value === log.saunaType);

  const details = [
    log.facilityName,
    setCount?.label,
    saunaType?.label,
  ]
    .filter(Boolean)
    .join(' / ');

  return [
    'サウナ行った。',
    `今日は${mood?.icon ?? ''} ${mood?.label ?? ''}。`,
    '',
    details,
    '',
    '#サウナ #ととのい',
  ].join('\n');
};
```

## 8. 保存仕様

## 8.1 MVPでの保存方法

MVPではローカル保存を想定する。

候補:

- AsyncStorage
- SQLite
- Expo SQLite

初期実装ではAsyncStorageを優先する。

## 8.2 保存するデータ

保存するのは `SaunaLog[]` の配列。

```ts
type SaunaLogs = SaunaLog[];
```

## 8.3 並び順

記録一覧は `date` または `createdAt` の新しい順で表示する。

## 9. 非対応機能

MVPでは以下の機能は実装しない。

- ログイン
- 会員登録
- クラウド同期
- 画像投稿
- サウナ施設検索
- 位置情報取得
- X APIによる自動投稿
- フォロー機能
- コメント機能
- グラフ表示
- カレンダー表示

## 10. 今後追加したい機能

MVP完成後に検討する。

- 施設名の履歴候補
- よく行く施設ランキング
- 月ごとのサウナ回数
- カレンダー表示
- ダークモード対応
- サウナ飯メモ
- 入浴料金メモ
- バックアップ機能
- iCloud同期
- 投稿文テンプレートの切り替え

## 12. UI方針

### デザイン方針

- 余白を多めにする
- 情報量を増やしすぎない
- 文字を読みやすくする
- ボタンを押しやすくする
- サウナ後でも迷わない画面にする

### 色の方向性

- 生成り
- 薄いグレー
- くすみベージュ
- 湯気っぽい白
- 淡い水色
- サーモンピンク

### 文言の方向性

- かたすぎない
- がんばらせない
- わかりやすい
- ちょっとゆるい

例:

- 書かなくてもOK
- 覚えてない
- よくわからん
- 記録したよ
- サウナ行った
